GOVERNED PROCESS

Governed Process

The evolving dialogue itself — user prompts, model outputs, turn history, and frame transitions. This process is dynamic, partially observed, and subject to disturbances including ambiguity, adversarial intent, and internal reasoning drift.

Every turn enters the runtime loop at this point. No output reaches the user without passing through the full diagnostic and control stack.

TELEMETRY

Structural Observation Module

The telemetry extraction stage. SOM measures and normalises observable structural signals from the live conversation stream — interaction pacing, coherence discontinuities, boundary pressure indicators, escalation patterns — and publishes a compact diagnostic state for downstream modules. NFV's adversarial context enriches this telemetry feed.

SOM has no decision authority. It does not interpret, decide, or act. Strict separation between measurement and modelling ensures diagnostic signals remain objective before stability analysis begins.

STATE ESTIMATION

Boundary-Coherence Structural Module

The structural stability model. BCSM evaluates stability across three dimensions: Core — identity continuity and invariant integrity; Boundary — the adaptive constraint envelope surrounding the interaction frame; Load — stress and pressure exerted on that envelope.

BCSM estimates coherence and deformation on the boundary-coherence manifold, flagging regime shifts such as boundary thinning, coherence loss, and instability under load. NFV's persistent risk scores and frame-escalation signals feed directly in, giving BCSM richer context than structural observables alone. Its output — structural condition — feeds forward to LTD-01 for trajectory classification.

NFV — Narrative Flow Validator

NFV operates as an upstream adversarial pre-wrapper — monitoring the full conversation for slow-burn jailbreak patterns, frame escalation, and trajectory risk before structural signals reach SOM and BCSM. It runs alongside the core kernel, not inside it, enriching both telemetry layers with adversarial context they would not otherwise have.

NFV maintains persistent adversarial suspicion scores, a frame taxonomy, semantic similarity gating, and trajectory risk tables across turns. Its graduated decisions — TEMPER · FULL_RECLASSIFY · SOFT_HALT · HARD_HALT — can act independently when risk thresholds are crossed, or feed enriched risk data forward into SOM and BCSM for more informed state estimation.

CLASSIFICATION

Latent Trajectory Dynamics

Using SOM telemetry and the BCSM stability assessment, LTD-01 classifies the aggregate diagnostic picture into one of four bounded trajectory states: Stabilising (trajectory converging toward coherence), Stalling (progress arrested, uncertainty accumulating), Degrading (coherence actively eroding), or Approaching Collapse (invariant failure imminent).

This bounded classification is passed to both the HBE (if epistemic pressure is detected) and the Stability Operator as the primary control input. LTD-01 cannot generate content, loosen constraints, or directly alter execution — its sole function is to provide a bounded diagnostic basis for regulatory action.

HBE — Humility Balance Engine

The HBE is an epistemic perturbation operator, not a pipeline stage. It activates when LTD-01 detects overconfident trajectories or when BCSM registers elevated ethical pressure. Rather than damping generation broadly, HBE specifically modifies confidence dynamics before stability control is applied.

When triggered, HBE encourages: clarification over assertion · uncertainty acknowledgement · reduced assertion strength · epistemic humility framing. It feeds its perturbation signal forward into the Stability Operator, allowing damping to be informed by epistemic as well as structural state.

Symbol Operator: 🐨 — Epistemic humility reflex (confidence dampening).

DAMPING CONTROL

Trajectory Damping Controller

The Stability Operator is a damping controller, not a safety gate. It receives LTD-01's trajectory classification and any perturbation signal from HBE, and attempts to restore conversational equilibrium before Guardian is invoked. This is the recovery mechanism — safety intervention is the last resort, not the first response.

When instability is detected, the Stability Operator may: reduce generation temperature · simplify response structure · prioritise clarification requests · acknowledge uncertainty explicitly · suppress unnecessary reasoning expansion. These standard decoding controls are activated dynamically by diagnostic state, not static configuration.

Symbol Operator: 🛟 — Trajectory stabilisation controller.

SAFETY GATE

Final Invariant Gate

Guardian is the final invariant enforcement gate in the runtime loop — it runs only after the Stability Operator has attempted recovery. It enforces structural invariants — Consent, Containment, Boundary Integrity, Transparency, Mutual Participation, and Spiral Lock — as hard safety gates. It is not a content filter and does not classify ethical correctness.

Guardian treats measurement uncertainty, instability, and disagreement as first-class safety signals. When boundaries become ambiguous, it fails closed — triggering containment or clarification rather than confident action. Operators: Pause/Clarify · Contain · Redirect · Disclose · Hard Halt.

Symbol Operator: 🛡️ — Invariant enforcement gate (fails closed).

EXECUTION

Constructive Reasoning Layer

If diagnostics indicate stability and all Guardian invariants remain intact, the constructive reasoning layer proceeds. This is where the system allows unconstrained model reasoning within the active frame contract.

The key architectural point: constructive reasoning is downstream of stability classification. This prevents the model's generative momentum from substituting for diagnostic safety. Symbol Code shapes how permitted reasoning unfolds — it is an execution policy layer, not the governance system itself.

FEEDBACK LOOP

Feedback & Evaluation Layer

The audit log records invariant checks and outcomes, diagnostic state values, trajectory classification events, operator selections, and boundary changes. This supports reproducibility, interpretability analysis, and governance traceability.

Metrics including SHI (Symbolic Harmony Index), RMT, ECE, and G-escape define a thresholded admissible operating region Ω. When SHI drops below the threshold band (≈ 0.93–0.96), control error signals feed back as offline governance tuning — threshold calibration and policy adjustment — not as real-time execution control.

Symbol Operator: ↓ — Traceability descent (forces low-ambiguity, traceable output).

FHP — Frame Handshake Protocol

A coordination protocol for multi-agent LLM systems that establishes explicit frame contracts, shared state continuity, and controlled frame transitions across interacting models. FHP extends the S-OS runtime loop across multiple AI agents without requiring fine-tuning of any model.

Demonstrated in a live cross-architecture session (Claude + ChatGPT, human router) — validating protocol compliance without weight modification. FHP operates alongside the core S-OS kernel, composing with it when multi-agent coordination is needed rather than replacing any internal module.
